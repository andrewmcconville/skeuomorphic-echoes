## Colors (Figma Variables)

All colors are stored as **Figma variables** (not color styles) and organized into five groups. Each variable is applied throughout the hi-fi prototype; the swatches below use the variables themselves.

### Primary Surface

| Variable | Hex | Usage |
|---|---|---|
| Surface-100 | #FAF7F2 | Card fills, modal backgrounds, lightest surface |
| Surface-200 | #F5F0E8 | Default page ground, body background |
| Surface-300 | #EDE8DC | Alternate ground for inset panels, warm bands |
| Surface-400 | #D8D3CA | Column dividers, borders, subtle separation |
| Surface-500 | #C8C2B8 | Disabled card fills, empty-state placeholders, muted text |

### Accent — Warm Overlay Spectrum

| Variable | Hex | Usage |
|---|---|---|
| Amber-500 | #E8A838 | Primary zone highlight on floor plan, active tab indicator, selected state |
| Amber-700 | #D4922A | Pressed/hover state for amber elements, progress bar fill |
| Orange-500 | #D46A38 | Secondary zone highlight, secondary category badges, era tags |
| Orange-700 | #C25A30 | Pressed/hover state for orange elements, active room indicator |

### Structure

| Variable | Hex | Usage |
|---|---|---|
| Ink-900 | #1A1A1A | Primary text, specimen borders, card strokes, button outlines |
| Ink-800 | #2A2A2A | Secondary headlines, navigation icons, pressed button fill |
| Gray-500 | #6B6B6B | Secondary text, metadata labels, scale marks, leader lines |
| Gray-400 | #8A8A8A | Tertiary text, placeholder copy, disabled labels |

### Semantic

| Variable | Hex | Usage |
|---|---|---|
| Collected-Green | #5A7A5A | Collected/completed state check icon, progress bar completed segment |
| Locked-Brown | #8A5A3A | Locked/unavailable state icon, restricted stage overlay |

| Variable | Value | Usage |
|---|---|---|
| Zone-Overlay-Opacity | 50% | Floor plan zone fill (amber or orange at half opacity over cream) |

## Typography (Figma Text Styles)

All text styles use the **IBM Plex** family — three optical roles that share a single design DNA. Every style below is saved as a Figma text style and applied throughout the hi-fi prototype.

### Display — IBM Plex Serif

| Style Name | Weight | Size / Line-Height | Usage |
|---|---|---|---|
| Display / Title | Bold | 80px / 88px | Exhibit title on home screen, room titles |
| Display / Specimen Name | Bold | 56px / 64px | Specimen name on artifact pages |
| Display / Heading 1 | Bold | 40px / 48px | Section headings, room landing introductions |
| Display / Heading 2 | Light | 32px / 40px | Sub-headings, card group labels, about screen title |

### Body / UI — IBM Plex Sans

| Style Name | Weight | Size / Line-Height | Usage |
|---|---|---|---|
| Body / Default | Regular | 24px / 36px | Room descriptions, one-line summaries, about bio |
| Body / Small | Regular | 20px / 28px | Expanded metadata paragraphs, cultural impact text |
| UI / Button Label | Medium | 20px / 20px | Button text, caps with +80 tracking |
| UI / Nav Label | Medium | 16px / 20px | Navigation labels, icon captions, back/close text |
| UI / Caption | Regular | 16px / 24px | Image credits, footnotes, source attributions |

### Metadata — IBM Plex Mono

| Style Name | Weight | Size / Line-Height | Usage |
|---|---|---|---|
| Meta / Era Tag | Regular | 18px / 24px | Era labels, uppercase with +100 tracking |
| Meta / Field Label | Regular | 14px / 20px | Catalog field keys (Medium, Dimensions, Weight) |
| Meta / Field Value | Light | 14px / 20px | Catalog field values, date ranges, stage position |

## Components (Figma Components)

All repeated elements are built as **Figma components** with variants where applicable. Instances are used throughout the hi-fi prototype; the main components live on the Research Page design system frame.

### Specimen Card

The primary content container — one per evolutionary stage. Feels like a pinned card on a museum wall.

| Variant | Description |
|---|---|
| Default | Cream fill (Surface-100), 2px Ink-900 stroke, 3px offset drop shadow, era tag in upper-left |
| Collected | Default + muted green (Collected-Green) check badge in upper-right corner |
| Locked | Surface-400 fill, dashed Ink-900 stroke, Locked-Brown lock icon, no tap action |

### Zone Overlay

Tappable regions on the floor plan map, one per exhibit room.

| Variant | Description |
|---|---|
| Unvisited | Amber-500 fill at Zone-Overlay-Opacity, 2px Ink-900 outline, drop shadow, room label centered |
| Visited | Surface-300 fill at 60% opacity, 1px Gray-500 outline, muted label |
| Active / Pressed | Orange-500 fill at 80% opacity, 2px Ink-900 outline, scale 0.97 feedback |

### Button

Dark-stroke bordered, cream-filled controls with caps labels. Minimum 88px touch target for kiosk input.

| Variant | State | Description |
|---|---|---|
| Primary | Default | Surface-100 fill, 2px Ink-900 stroke, label in UI / Button Label style |
| Primary | Hover | Ink-800 fill, Surface-100 text (inverted) |
| Primary | Disabled | Surface-400 fill, Gray-400 stroke and text, no tap action |
| Secondary | Default | No fill (transparent), 1px Gray-500 stroke, Gray-500 label |
| Secondary | Hover | Surface-300 fill, Ink-900 stroke and label |

### Header

Persistent top navigation bar — minimal chrome so the specimen content stays in focus.

| Variant | Description |
|---|---|
| Main Header | Back arrow + exhibit title + floor map icon + collection counter; dotted leader lines connecting icon to label |
| Slim Header | Back/close icon only; used on interior specimen pages where full nav would compete with dense metadata |

### Artifact Card

Used on the Artifact Collection screen — one card per collected evolutionary stage.

| Variant | Description |
|---|---|
| Collected | Cream fill, 2px Ink-900 stroke, artifact name (Display / Heading 2), era tag, thumbnail, grouped under skeuomorph room heading |
| Locked | Surface-400 fill, dashed stroke, silhouette placeholder for uncollected artifact |

### Toast / Confirmation

Triggered when a visitor taps "Collect" on a specimen page.

| Variant | Description |
|---|---|
| Collectible Confirmation | Surface-100 fill, Ink-900 stroke, Collected-Green check icon, specimen name + "Added to collection" message, auto-dismiss after 3 seconds |

### Era Tag

Inline label identifying a specimen's historical period.

| Variant | Description |
|---|---|
| Default | Surface-300 pill, Meta / Era Tag text style, e.g. "Mechanical · c. 1450" |
| Active | Orange-500 pill, Surface-100 text, used when the visitor is viewing that era |

## Design System Usage

### On the design system frame (Research Page)

- Color swatches use the Figma color variables listed above — changing a variable updates every swatch and every instance across the file.
- Typography samples use the Figma text styles — each sample shows the style name, typeface, weight, size, and line-height.
- Component instances are placed beside their descriptions; variants are shown side-by-side for comparison.

### Across the hi-fi prototype

- Every screen uses color variables for fills, strokes, and text — no hard-coded hex values.
- Every text layer references a text style — no unlinked type.
- Every repeated element is an instance of a main component — edits propagate automatically.
- Browsing screens (home, floor plan, artifact collection) use lighter weights, warmer accents, and sparser density.
- Detail screens (specimen interiors, metadata deep-dives) use heavier weights, near-black dominance, and tighter spacing.

> The design system does not need to be applied to the Research Page itself or the Lo-fi Wireframe Page.
