## Content Inventory

### Home Screen — Floor Plan Map

> **Register: Warm / Gallery** — cream ground, amber/orange overlays, lighter type weights, sparse and inviting

| Content | Type | Notes |
|---|---|---|
| Title ("Before the Prompt") | Text | IBM Plex Serif Bold, large; anchors exhibit identity on cream ground |
| Subtitle / tagline | Text | IBM Plex Sans Regular; one-line hook explaining the exhibit concept |
| Floor plan illustration | Image / interactive map | Top-down layout of the four rooms; primary navigation surface |
| Room zones (×4) | Tappable regions with labels | Amber/orange semi-transparent fills (40–60% opacity), 2px near-black outline, 88px min touch targets |
| Zone state indicators | Color shift | Visited zones mute toward warm gray; unvisited stay vibrant amber |
| Visited Rooms icon + count | Icon + text | Minimal chrome; shows stages collected out of total |
| About icon | Icon | Links to credits, sources, and project info |
| Collection progress indicator | Icon + text/badge | Overall completion across all four rooms |

### Visited Rooms — List Screen

> **Register: Warm / Gallery** — overview and collection tracking; sparse, card-based layout

| Content | Type | Notes |
|---|---|---|
| Title ("Visited Rooms") | Text | IBM Plex Serif; consistent heading hierarchy |
| Back / Home navigation | Icon button | Back arrow returns to floor plan; persistent minimal chrome |
| Room cards (×4) | Card: room name + thumbnail + progress | Cream fill, 2–3px near-black stroke, offset drop shadow; tap to jump to room |
| Collected stages per room | Text / badge count | IBM Plex Mono; e.g., "3 / 5 collected" beneath each card |
| Stage thumbnails (collected) | Image grid or list | Small specimen previews for stages already collected |
| Empty/locked state indicators | Icon + text | Muted red-brown for locked; grayed-out placeholder for unvisited |

### About Screen

> **Register: Warm / Gallery** — informational, low density; colophon-style credits block

| Content | Type | Notes |
|---|---|---|
| Title | Text | "About" or exhibit title |
| Subheading | Text | Brief descriptor of the exhibit's theme |
| Navigation (back/home) | Icon buttons | Back arrow to return to floor plan |
| Photo / portrait | Image | Creator headshot or exhibit logo |
| Bio | Text | Short paragraph about the designer/curator |
| Contact info | Text / email link | Email or portfolio link for the creator |
| Project overview | Text | What the exhibit covers and why it exists |
| Sources | | Colophon-style section; dense mono-set key: value block |
| - Works cited | Text + URL | Bringhurst, Letterform Archive, CHM, etc. |
| - Image credits | Text + URL | Photo and illustration attributions |
| - Figma plugins | Text + URL | Tools used in the design process |

### Room Landing (×4)

> **Register: Transitional** — warm ground with emerging catalog density; the threshold between gallery browsing and specimen study

| Content | Type | Notes |
|---|---|---|
| Room title (e.g., "WYSIWYG") | Text | IBM Plex Serif Bold; names the skeuomorph category |
| Room description / intro | Text | 2–3 sentences; IBM Plex Sans — what physical object persists and where |
| Skeuomorph summary | Text | One-line thesis connecting the original artifact to its digital echo |
| Navigation (back to map, visited rooms) | Icon buttons | Persistent top bar; minimal chrome with dotted leader lines |
| Evolutionary stage cards (4–5 per room) | Card: title + thumbnail + era tag | Specimen-card style: cream fill, dark stroke, drop shadow; shows collected state |
| Collection progress for this room | Progress bar / badge | Muted green fill for collected; visual bar showing room completion |
| Floor plan mini-map (current position) | Small image / icon | Highlights which room the visitor is currently in |

### Interior Page — Evolutionary Stage (×20)

> **Register: Cool / Clinical** — cream + near-black + mid-gray dominant, heavier type weights, tighter spacing, dense and authoritative. The visitor is now studying the artifact. Layout reads like a museum catalog raisonné or a specimen tag pinned beside a display case.

| Content | Type | Notes |
|---|---|---|
| **Core fields (always present)** | | |
| Specimen name | Text (display) | IBM Plex Serif Bold; largest text element on the page |
| Era tag | Text (mono) | IBM Plex Mono uppercase; e.g., "Mechanical · c. 1450" |
| Date range | Text (mono) | e.g., "c. 1450 – c. 1880" |
| Stage position | Text (mono) | e.g., "3 of 6"; locates user in the evolutionary chain |
| One-line summary | Text (body) | IBM Plex Sans; 1–2 sentences max describing the artifact |
| Hero specimen illustration | Image | Centered on cream ground, generous padding; largest visual element |
| Navigation (previous / next / close / back to room) | Icon buttons | Tappable lineage links; swipe or tap to move through the chain |
| **Catalog fields (select 6–10 per stage)** | | Dense mono-set key: value column, separated by thin rules |
| *Physical properties* | | |
| - Medium / Material | Text (mono) | e.g., "Lead alloy, oak frame, iron chase" |
| - Dimensions | Text (mono) | e.g., "220 × 45 × 30 mm" |

| *Mechanism* | | |
